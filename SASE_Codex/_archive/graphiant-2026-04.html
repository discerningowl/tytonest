Bottom Line Up Front

Architecture: Layer 3 Private Network Without Tunnels
The "tunnel tax" problem and why Graphiant's model is architecturally distinct

The Tunnel Tax Problem

Traditional SD-WAN builds IPsec tunnels between branch appliances and cloud PoPs. This adds overhead: IPsec key negotiation latency on startup, encryption/decryption processing on every packet, and management complexity from maintaining tunnel state across hundreds of sites. For most enterprise WAN use cases, this overhead is acceptable. For latency-deterministic workloads — financial trading, industrial control systems, real-time unified communications — the tunnel overhead and jitter introduce measurable performance problems.

Graphiant's model eliminates tunnels entirely. The customer's existing edge router sends traffic to the nearest Graphiant edge node via standard internet. From there, Graphiant's private network fabric routes traffic internally to the destination site's Graphiant edge node. The enterprise sees a private Layer 3 connection; Graphiant's network handles routing and transport. No tunnels to configure, no IPsec state to maintain, no rekeying events to manage.

THE TUNNEL-FREE MODEL

Customer edge router → internet → Graphiant edge node → Graphiant private fabric → destination Graphiant edge node → customer edge router. The customer's infrastructure sees a private routed connection. Graphiant's fabric handles SLA enforcement, path steering, and transport optimization internally.

WHAT GRAPHIANT IS NOT
  • Not a full SASE platform — no SSE, no ZTNA, no AIOps
  • Not a security service — no DLP, no SWG, no threat inspection
  • Not replacing a customer's edge firewall or router
  • Not competing with Cato, Netskope, or Cloudflare on security
  • Competing on the WAN underlay layer with MPLS and SD-WAN overlays

Pillar 3 — SD-WAN (Connectivity Scope)
Layer 3 private connectivity · SLA-backed latency · no tunnel overhead

SD-WAN Analysis

Graphiant is evaluated on SD-WAN connectivity criteria where its tunnel-free model is directly relevant. It does not score on security convergence, SSE integration, or CPE WAN optimization features (FEC, packet duplication) because these are outside its architectural scope.

Private Connectivity

Graphiant's network fabric provides private Layer 3 connectivity with SLA-backed latency guarantees. Traffic between sites never traverses public internet routing once it enters the Graphiant fabric — routing decisions are made within Graphiant's network, not by BGP path selection on the public internet. This provides more deterministic latency than internet-based SD-WAN overlays and comparable predictability to MPLS for latency-sensitive workloads.

Target Use Cases

Graphiant targets organizations with specific deterministic performance requirements: financial trading (sub-millisecond latency variance), industrial control systems (deterministic control loop timing), healthcare real-time imaging (PACS/DICOM transfer performance), and unified communications (voice and video jitter sensitivity). These are workloads where IPsec tunnel overhead and BGP path variability are measurable problems, and where MPLS retirement requires a replacement that maintains deterministic behavior.

Security Stack Integration

Graphiant does not provide a security stack. Organizations deploying Graphiant must bring their own security via service chaining at the Graphiant edge node — the Graphiant edge node can forward traffic to a co-located Big Five security service for inspection before delivery. The integration model requires network architecture planning; it is not a plug-and-play security replacement.

▲ Strengths

Eliminates tunnel management overhead and IPsec complexity. SLA-backed Layer 3 latency guarantees — closer to MPLS predictability than internet SD-WAN. Consumption-based pricing model. Simpler operations than SD-WAN overlay architectures. Right for MPLS retirement in performance-sensitive verticals.

▼ Watch Areas

No SSE, no ZTNA, no security stack — must be paired with a separate security vendor. Limited PoP coverage vs. mature SD-WAN vendors. Production validation at enterprise scale limited as of Q2 2026. Not suitable for greenfield SASE deployments where security convergence is a requirement. Reassess Q3 2026.
